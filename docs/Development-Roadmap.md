## P0 - 무조건 개발 (Critical Path)

핵심 사용자 경험에 직결되는 기능. 이 기능들 없이는 제품 가치가 심각하게 훼손됨.

### 1. Career 탭 - 이력서 추출 품질 개선 ⭐⭐⭐
**현재 문제**: 이력서 파싱 정확도 ~50%, 실제 활용 불가 수준

| 항목 | 설명 |
|------|------|
| **문제점** | PDF 파싱 시 레이아웃/포맷에 따른 텍스트 추출 실패, OCR 정확도 낮음, 경력/프로젝트 구분 오류 |
| **개선 방향** | - 다양한 이력서 템플릿 테스트 케이스 확보<br>- parse-resume 프롬프트 고도화<br>- 사용자 수동 교정 UI 강화<br>- 추출 실패 시 fallback 로직 (수동 입력 유도) |
| **성공 지표** | 이력서 추출 정확도 85% 이상 |
| **예상 공수** | 3-5일 |
| **🔍 조사 필요** | 외부 서비스 연동 검토 (Affinda, Sovren 등) - 자체 개선 vs 외부 API 비용/정확도 비교 분석 필요 |

### 2. Career 탭 - 맞춤 이력서 생성 품질 개선
**현재 문제**: 생성된 맞춤 이력서의 실용성/완성도 검증 필요

| 항목 | 설명 |
|------|------|
| **개선 방향** | - generate-resume 프롬프트 고도화<br>- 공고별 키워드 매칭 강화<br>- 이력서 미리보기 UI 개선<br>- 다양한 포맷 지원 (현재 DOCX만) |
| **성공 지표** | 사용자 만족도 조사, 생성 → 다운로드 전환율 |
| **예상 공수** | 2-3일 |

### 3. Chat 탭 - 공고 분석 정확도 개선
**현재 문제**: 비공고 URL도 경고 후 보드에 추가 가능

| 항목 | 설명 |
|------|------|
| **개선 방향** | - 공고 여부 판별 로직 강화 (analyze-job 프롬프트)<br>- 신뢰도 점수 기반 자동 차단 (threshold 설정)<br>- 사용자에게 "이 URL이 채용공고가 맞나요?" 확인 단계 추가 |
| **성공 지표** | 비공고 오등록률 < 5% |
| **예상 공수** | 1-2일 |

### 4. UI/UX - 데스크톱 최적화
**현재 문제**: 모바일 퍼스트로 개발, 데스크톱 레이아웃 개선 필요

| 항목 | 설명 |
|------|------|
| **개선 방향** | - 넓은 화면 활용 레이아웃<br>- 사이드바 네비게이션<br>- 멀티 패널 뷰 (보드 + 상세 동시 표시)<br>- 키보드 단축키 |
| **예상 공수** | 3-4일 |

## P1 - 되면 좋음 (High Value)

제품 완성도와 사용자 경험을 크게 향상시키는 기능.

### 1. Goals 탭 - 진행률 트래킹
**현재 문제**: 목표 대비 진행 상황 시각화 없음

| 항목 | 설명 |
|------|------|
| **개선 방향** | - 지원 현황 기반 자동 진행률 계산<br>- 상태별 통계 (서류지원 N건, 인터뷰 N건, 오퍼 N건)<br>- 목표 달성률 대시보드<br>- 주간/월간 활동 리포트 |
| **계산 로직 예시** | 목표: 인터뷰 5건 → 현재 인터뷰 상태 3건 = 60% 달성 |
| **예상 공수** | 2-3일 |

### 2. Board 탭 - 고급 기능
**현재 문제**: 메모, 태그 등 관리 기능 없음

| 항목 | 설명 |
|------|------|
| **추가 기능** | - 공고별 메모 기능<br>- 태그/라벨 시스템<br>- 면접 일정 등록<br>- 알림/리마인더 |
| **예상 공수** | 3-4일 |

### 3. Settings - 구독 관리 완성
**현재 문제**: 플랜 변경, 취소 등 관리 기능 미구현

| 항목 | 설명 |
|------|------|
| **필수 기능** | - 현재 구독 상태 표시<br>- 플랜 업/다운그레이드<br>- 구독 취소 및 재개<br>- 결제 수단 변경<br>- 크레딧 사용 내역 |
| **예상 공수** | 2-3일 (결제 연동 후) |

### 4. 결제 시스템 실연동
**현재 문제**: UI만 존재, 실제 결제 불가 (20% 완성)

| 항목 | 설명 |
|------|------|
| **필수 기능** | - Stripe 연동 (구독 결제)<br>- 플랜별 기능 제한 적용<br>- 결제 실패/취소 처리<br>- 영수증/인보이스 발행 |
| **성공 지표** | 결제 성공률 > 95%, 구독 전환 가능 |
| **예상 공수** | 5-7일 |

### 5. 소셜 로그인
**현재 문제**: 이메일 로그인만 지원

| 항목 | 설명 |
|------|------|
| **추가 기능** | - Google 로그인<br>- Apple 로그인 (iOS 사용자용) |
| **예상 공수** | 1-2일 |

## P2 - 되면 괜찮음 (Nice to Have)

제품 차별화 및 고급 기능. 핵심 기능 안정화 후 진행.

### 1. Career 탭 - 이력서 포맷 다양화

| **예상 공수** | 2-3일 |

| 항목 | 설명 |

| **예상 공수** | 2-3일 |

### 3. 고급 보안 기능

| 항목 | 설명 |

| **예상 공수** | 2-3일 |

### 4. UI/UX - 접근성 개선

| 항목 | 설명 |
|------|------|
| **개선 방향** | - WCAG 2.1 AA 준수<br>- 스크린 리더 지원<br>- 고대비 모드<br>- 키보드 네비게이션 완성 |
| **예상 공수** | 2-3일 |

### 5. 알림 시스템

| 항목 | 설명 |
|------|------|
| **기능** | - 이메일 알림 (면접 일정, 마감일)<br>- 푸시 알림 (PWA)<br>- 알림 설정 커스터마이징 |
| **예상 공수** | 3-4일 |

### 6. 데이터 동기화

| 항목 | 설명 |
|------|------|

| **예상 공수** | 4-5일 |

## 개발 우선순위 타임라인 (권장)

```
Phase 1 (Week 1-2): P0 집중
├── 이력서 추출 품질 개선 (3-5일)
├── 공고 분석 정확도 개선 (1-2일)
└── 맞춤 이력서 생성 개선 (2-3일)

Phase 2 (Week 3-4): P0 완료 + P1 시작
├── 결제 시스템 실연동 (5-7일)
└── Goals 진행률 트래킹 (2-3일)

Phase 3 (Week 5-6): P1 완료
├── Board 고급 기능 (3-4일)
├── Settings 구독 관리 (2-3일)
└── 소셜 로그인 (1-2일)

Phase 4 (Week 7+): P2 순차 진행
├── 데스크톱 최적화
├── 이력서 포맷 다양화
└── 기타 고급 기능
```

## 의사결정 필요 사항

### 1. 결제 시스템
Stripe 단독 vs 국내 PG 병행?

### 2. 이력서 파싱 ⚠️ 조사 진행 중
**자체 개선 vs 외부 서비스 연동**

| 옵션 | 장점 | 단점 | 비용 |
|------|------|------|------|
| **자체 개선** | 비용 없음, 커스터마이징 자유 | 개발 시간 소요, 정확도 한계 | 개발 공수 3-5일 |
| **Affinda** | 높은 정확도 (90%+), 다양한 포맷 지원 | API 비용, 외부 의존성 | $0.10-0.50/건 |
| **Sovren** | 엔터프라이즈급 정확도, ATS 통합 | 고비용, 복잡한 설정 | 월 $500+ |
| **Eden AI** | 여러 파서 통합, 가격 경쟁력 | 정확도 가변적 | $0.05-0.20/건 |

**권장**: 먼저 자체 개선 시도 후, 정확도 80% 미달 시 Affinda 검토

### 3. 데이터 저장
localStorage 유지 vs 전면 Supabase 마이그레이션?

### 4. 타겟 시장
국내 중심 vs 글로벌 (다국어 지원 우선순위에 영향)
